We want to build a complex new feature that adds custom GeoJSON to the map. In the builder we should create a PropGeoJSON base class that lets a user paste the URL to a GeoJSON file and choose a type. The types will be "areas", "lines", and "points". Each type will be rendered by a different Feature in CustomGlobe. We should add an interface in the builder that lets us show and hide geojson features based on geojson property. The user can choose which geojson property to filter based on, then choose which values to show (default all) with a typeahead component from components-builder. Additionally we should have a colour-scale feature that allows the user to pick a property, set a min & max value, and each visible feature will be coloured based on a colour scale. We want the ability to animate between configs, for instance to change the property used to calculate the colour scale, so we should be careful in Svelte state to handle the lifecycle correctly.

The render methods for these types will be able to be swapped out, potentially configured in the builder. For the first implementation we want areas and lines to be implemented in MapLibre layers, and we want points to be implemented as DOM elements with CSS transforms and transitions. Points should additionally be implemented using the ThreeJS spike layer with the height determined by the scale. In future we will implement an alternative 3d line implementation, so keep extensibility in mind.

The areas and lines implementation shares a lot of similarities with the HighlightCountries feature and may be able to share componentised code - let's identify and see what's reusable, and if it would be easier to keep the implementations separate.
